# Antigravity Quota Monitor (agy-quota-monitor)

Windowsのタスクバー（システムトレイ）常駐型の Antigravity（IDE, デスクトップアプリ2, SDK）クォータ監視アプリケーションです。
最も残量の少ないAIモデルの残量パーセンテージをトレイアイコンに動的に描画し、視覚的に知らせます。

---

## 主な機能

* **動的トレイアイコン表示**: 最低残量モデルのクォータ割合を数値アイコン（例: `67` など）で表示。残量レベルに応じて色変化（緑 ➔ 黄 ➔ 赤 ➔ 暗赤）します。
* **マウスホバー（ツールチップ）**: 各モデルの残量・リセットまでの時間、および月間クレジット残量をポップアップで確認できます。
* **コンテキストメニュー（右クリック）**:
  * **全モデル詳細表示 (Show Details)**: クォータ制限のある全モデルとリセット時間の一覧表示。
  * **今すぐ更新 (Refresh Now)**: 即時ポーリング。
  * **表示対象の切り替え (Display Model)**: アイコン表示する優先モデルを個別指定（または「自動」）。
  * **接続先インスタンスの切り替え (Select Instance)**: IDEとデスクトップアプリが同時起動している場合に、接続先PIDをメニューから手動切り替え。
  * **更新間隔の変更 (Polling Interval)**: 5分 / 10分 / 15分 / 30分 から選択可能。

## 🔒 セキュリティとプライバシーの開示

本アプリケーションを公開し配布するにあたり、技術者倫理に基づき、セキュリティに懸念を持たれやすい挙動についてあらかじめ技術仕様を開示します。

### 1. 挙動と仕組みについて
本アプリは、Antigravity の言語サーバーが起動するローカルの Connect RPC HTTPS API を呼び出してデータを取得しています。
* **プロセスの探索**: システム上で実行されている `language_server.exe` または `language_server_windows_x64.exe` のコマンドライン引数を WMI/PowerShell で取得します。
* **CSRFトークンの取得**: コマンドライン引数に含まれる `--csrf_token` から認証用トークンを抽出します。
* **ポートの動的検出**: コマンドライン引数にポート番号がない場合（ランダムポート割り当て時）、該当PIDがリスニングしているTCPポートを列挙し、HTTPS POSTによるヘルスチェックでAPIポートを特定します。

> [!NOTE]
> 認証情報（CSRFトークン）を他プロセスから抽出する設計ですが、これはローカルでランダムに割り当てられるポートのAPIに接続するための**唯一の実装手段**です。

### 2. テレメトリ（個人情報の収集）の排除
本アプリは**一切のテレメトリ（利用状況の送信、エラーレポート、情報収集機能）を搭載していません。**
通信はローカルホスト（`127.0.0.1`）上の Antigravity 言語サーバーのみと行われます。インターネット上の外部サーバーへの通信は一切発生しません。

### 3. ウイルス対策ソフトの誤検知（False Positive）について
「別プロセスの引数読み取り」「コマンドラインでのWMI実行」「SSL証明書検証を無効化したローカルHTTPS接続」といった挙動は、セキュリティソフト（Windows Defenderなど）によってマルウェア（情報窃取ツール）と誤検知される場合があります。
不審なバイナリでないことを担保するため、リリース実行ファイル（`.exe`）はローカルでビルドせず、**GitHub Actionsのクリーンなクラウド環境で自動ビルドされたもの**を Release に添付しています。

## 🚀 使い方

### 方法A：GitHub Actionsビルド版を利用する (推奨)
1. リポジトリの [Releases](https://github.com/atollray/agy-quota-monitor/releases) ページから最新の `agy-quota-monitor.exe` をダウンロードします。
2. ダブルクリックして実行すると、タスクバーの通知領域（システムトレイ）に起動します。

### 方法B：Python環境から直接実行する (最もセキュア)
実行ファイルを信用せず、ソースコードから直接実行したい場合は以下を実行します。

1. Python 3.10以上がインストールされていることを確認します。
2. 依存関係をインストールします。
   ```bash
   pip install -r requirements.txt
   ```
3. アプリケーションを実行します。
   ```bash
   python src/main.py
   ```

### 方法C：自分でビルドする (セルフビルド)
自分でコードを修正し、スタンドアロンの `.exe` を作成したい場合：

1. 開発用の依存関係がインストールされていることを確認し、以下を実行します。
   ```bash
   pyinstaller --onefile --noconsole --name agy-quota-monitor src/main.py
   ```
2. ビルド成功後、`dist/agy-quota-monitor.exe` に実行ファイルが生成されます。
